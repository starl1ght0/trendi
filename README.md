# TrendChartsForArchivarius
Графики трендов для архиватора

## Быстрый запуск (portable, для переноса на другой ПК)

1. **Экспорт данных из PostgreSQL** (один раз, на текущем компьютере):
   ```bash
   npm install
   # в .env укажите параметры PostgreSQL (см. .env.example)
   npm run export-db
   ```
   Данные сохранятся в `data/trends.db`.

2. **Сборка portable-версии**:
   ```bash
   npm run build:portable
   ```

3. **Перенос**: скопируйте папку `dist/TrendChartsPortable` целиком на флешку или другой компьютер.

4. **Запуск**: дважды щёлкните `TrendCharts.exe` — откроется браузер с графиками.

В portable-версии уже есть Node.js и база `data/trends.db`. Ничего дополнительно устанавливать не нужно.

## Разработка

```bash
npm install
cp .env.example .env
npm run launch   # сервер + браузер
# или
npm start        # только сервер
```

База по умолчанию: `data/trends.db` (SQLite, один файл — легко копировать).

## ТЗ

Отдельная страница для просмотра графиков трендов. Обыкновенный xy граф, белый фон, динамический компонент. Получает инфу запросом к серверу (развернут там же, где и фронт). `/api/trends/?x=[{dt_interval}]&y={col}&online={bool}`
Графики получаем, рисуем. Занимать всю доступную область, полностью резиновый фронт, если WS - свитчим протокол с http на ws, эндпоинт такой же. Хедер, другие панели скину позже, референсы позже. Пока просто подготовь форму и сам себе замокай сервер. Данных много, все отображать. Добавить возможность настроить график - подписи осей, масштаб, цвета линий. 

## ОТ МЕНЯ

тестовая бд выглядит как:

send time | id | value | execution_time_ms

Основной функционал:

* Запрос данных с сервера через API /api/trends.

* Поддержка двух колонок для отображения: "value" и "execution_time_ms". (если что переименую, не знаю контекста какие именно колонки должны быть)

* Ползунок: если данных больше 30 точек, появляется скроллбар. Позволяет плавно прокручивать окно отображения.

* Режим онлайн: при включённом флажке график автоматически обновляется каждые 3 секунды, запрашивая данные с текущей конечной датой. При ручном скролле или переключении колонки онлайн автоматически отключается.

* Настройка цвета линии графика: доступно 5 предустановленных цветов.

* При наведении на точки: показывают значения выбранной колонки, время точки (из send_time), execution_time_ms и value.

* Кнопка "Настройки": смена цвета, выбор промежутка даты, выбор колонки.

* Данные получаются с сервера, который обращается к SQLite. Если база данных недоступна, приложение покажет ошибку.

* График полностью резиновый, занимает всю доступную область, при изменении размера окна перерисовывается.
